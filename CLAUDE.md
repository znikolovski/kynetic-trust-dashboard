## Design Context

This project uses the **Kinetic Trust Dashboard** design system (register: product).

- `PRODUCT.md` — strategic context: register, users, positioning, anti-references, design principles
- `DESIGN.md` — visual system spec: color tokens, typography, elevation, components, Do's and Don'ts
- `.impeccable/design.json` — sidecar: rendered component snippets, tonal ramps, shadow/motion tokens, full narrative

Read all three before generating new screens, panels, or components. The frontmatter YAML in `DESIGN.md` is normative for token values; prose contextualizes how to apply them. The North Star is **"The Neon Vault"** — the dashboard is the interior of the vault: authenticated, live, institutionally precise.

Key product constraints: WCAG 2.1 AAA throughout; skeleton screens not spinners on data surfaces; status vocabulary is closed (COMPLETED / SETTLED / PENDING); JetBrains Mono for every numeric value.
